## 简介

OFlatNas 是一个轻量级、高度可定制的个人导航页与仪表盘系统。它基于 Vue 3 与 Go(Gin) 构建，旨在为 NAS 用户、极客和开发者提供一个优雅的浏览器起始页。

### 核心特性
- **多端统一入口**：把常用网站、内网服务和工具聚合在同一仪表盘
- **文件与媒体能力**：内置文件传输助手、音乐播放器和壁纸管理
- **内外网智能切换**：自动识别网络环境并路由到最佳地址
- **本地数据可控**：配置与数据存储在本地目录，迁移与备份更方便
- **可视化组件生态**：内置多种组件，支持自定义 CSS/JS 深度扩展
- **资源内存占用极低**：NAS端占用100MB内存，访问端真实内存占用不到80兆

## 环境要求

### 系统要求
- **Docker 环境**：Docker Engine 20.10+
- **Debian/Ubuntu 系统**：Debian 10+ 或 Ubuntu 18.04+
- **硬件要求**：
  - CPU：最低 1 核心
  - 内存：最低 128MB（推荐 512MB+）
  - 存储：至少 1GB 可用空间

### 网络要求
- **端口开放**：
  - 前端访问端口：默认 23000（可自定义）
  - 后端服务端口：默认 3000（可自定义）
- **防火墙配置**：确保上述端口在防火墙中开放

### 软件依赖
- **Docker 方式**：无需额外依赖
- **Debian/Ubuntu 方式**：系统会自动安装 Nginx、curl、iproute2 等必要工具

## 部署方式对比

| 部署方式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|----------|
| Docker 部署 | 部署简单、环境隔离好、版本管理方便 | 需要 Docker 环境、资源占用相对较高 | 生产环境、容器化部署 |
| Debian/Ubuntu 一键部署 | 性能最优、完全控制、适合本地部署 | 需要 Linux 系统、配置相对复杂 | 服务器环境、需要深度定制 |
| 本地安装（Release 包） | 最简单、无需依赖 | 仅支持 Linux、功能相对受限 | 开发测试、快速体验 |
| Docker Compose | 配置灵活、便于管理 | 需要 Docker Compose 环境 | 开发环境、多服务编排 |

## Docker 部署

### 基础部署

```bash
# 拉取官方镜像
docker pull qdnas/flatnas:latest

# 创建数据目录
mkdir -p ./data ./doc ./music ./PC ./APP

# 启动容器
docker run -d \
  --name flatnas \
  -p 23000:3000 \
  -v $(pwd)/data:/app/server/data \
  -v $(pwd)/doc:/app/server/doc \
  -v $(pwd)/music:/app/server/music \
  -v $(pwd)/PC:/app/server/PC \
  -v $(pwd)/APP:/app/server/APP \
  -v /var/run/docker.sock:/var/run/docker.sock \
  qdnas/flatnas:latest
```

### 环境变量配置

```bash
# 设置时区
docker run -d \
  --name flatnas \
  -e TZ=Asia/Shanghai \
  -p 23000:3000 \
  -v $(pwd)/data:/app/server/data \
  qdnas/flatnas:latest
```

### 数据卷说明

| 数据卷 | 用途 | 默认路径 |
|--------|------|----------|
| `/app/server/data` | 配置文件和用户数据 | `./data` |
| `/app/server/doc` | 文件传输助手数据 | `./doc` |
| `/app/server/music` | 音乐文件 | `./music` |
| `/app/server/PC` | 桌面壁纸 | `./PC` |
| `/app/server/APP` | 移动端壁纸 | `./APP` |
| `/var/run/docker.sock` | Docker 容器管理 | `/var/run/docker.sock` |

## 本地安装（Release 包）

### 手动部署步骤

```bash
# 1. 下载 Release 包
wget https://github.com/Garry-QD/FlatNas/releases/latest/download/release.zip

# 2. 解压到指定目录
unzip release.zip -d /opt/flatnas

# 3. 赋予执行权限
chmod +x /opt/flatnas/flatnas-server

# 4. 启动服务
cd /opt/flatnas
./flatnas-server
```

### 访问地址

- **默认访问地址**：`http://<服务器IP>:3000`
- **管理界面**：`http://<服务器IP>:23000`

## Docker Compose 部署

### 配置文件

```yaml
version: '3.8'

services:
  flatnas:
    image: qdnas/flatnas:latest
    container_name: flatnas
    restart: unless-stopped
    ports:
      - '23000:3000'
    volumes:
      - ./data:/app/server/data
      - ./music:/app/server/music
      - ./PC:/app/server/PC
      - ./APP:/app/server/APP
      - ./doc:/app/server/doc
      - /var/run/docker.sock:/var/run/docker.sock
```

### 启动命令

```bash
# 启动服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f
```

## 常见问题排查

### 端口冲突

**问题**：端口已被占用
```bash
# 检查端口占用
lsof -iTCP:23000
netstat -tulpn | grep 23000

# 修改端口配置
sudo nano /etc/flatnas/flatnas.env
```

### 权限问题

**问题**：文件权限错误
```bash
# 检查目录权限
ls -la /opt/flatnas/

# 修复权限
sudo chown -R flatnas:flatnas /opt/flatnas/
sudo chmod -R 755 /opt/flatnas/
```

### Docker 权限问题

**问题**：Docker Socket 权限不足
```bash
# 添加用户到 docker 组
sudo usermod -aG docker $USER

# 重新登录或重启系统
```

### 日志查看

```bash
# 查看应用日志
journalctl -u flatnas -n 50 -f

# 查看 Docker 容器日志
docker logs -f flatnas

# 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log
```

### 性能优化

**Docker 环境**：
```bash
# 设置资源限制
docker run -d \
  --memory=512m \
  --cpus=1.0 \
  -p 23000:3000 \
  qdnas/flatnas:latest
```

**Debian 环境**：
```bash
# 调整 systemd 服务配置
sudo nano /etc/systemd/system/flatnas.service

# 重启服务
sudo systemctl daemon-reload
sudo systemctl restart flatnas
```

## 性能优化建议

### 硬件优化

1. **内存配置**
   - 建议至少 512MB RAM
   - 避免同时运行过多容器

2. **存储优化**
   - 使用 SSD 存储提升读写速度
   - 合理规划数据卷大小

### 网络优化

1. **CDN 加速**
   - 配置 CDN 缓存静态资源
   - 启用 Gzip 压缩

2. **负载均衡**
   - 多实例部署
   - 使用反向代理集群

### 安全配置

1. **HTTPS 部署**
   - 配置 SSL 证书
   - 启用强制 HTTPS

2. **访问控制**
   - 配置防火墙规则
   - 启用访问日志审计

### 监控告警

1. **系统监控**
   - 监控 CPU、内存、磁盘使用率
   - 设置告警阈值

2. **应用监控**
   - 监控 API 响应时间
   - 监控数据库连接数
